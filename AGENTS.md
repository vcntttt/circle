# AGENTS.md

## Commands

- Use `pnpm` only. The lockfile is `pnpm-lock.yaml`.
- Dev server: `pnpm dev`
- Production build: `pnpm build`
- Production server: `pnpm start`
- Lint: `pnpm lint`
- Format whole repo: `pnpm format`
- Drizzle: `pnpm db:generate`, `pnpm db:migrate`, `pnpm db:seed`, `pnpm db:studio`

## Verification

- There is no test suite and no `typecheck` script.
- The meaningful verification path is `pnpm build && pnpm lint`.
- Pre-commit only runs `npx lint-staged`, which formats staged files with Prettier.
- `src/routeTree.gen.ts` is generated by TanStack Router during route builds/dev and is ignored by Prettier. Expect it to change when editing routes.

## Architecture

- This is a single-package TanStack Start app, not a monorepo.
- Runtime entrypoints are `src/client.tsx`, `src/router.tsx`, and `src/routes/__root.tsx`.
- Canonical app routes are `/projects`, `/issues`, `/issues/$issueIdentifier`, and `/settings`.
- `src/routes/$orgId*` files are legacy compatibility redirects only.
- Shared app shell lives in `components/layout/main-layout.tsx` and `components/layout/sidebar/*`.
- Server mutations/loaders live in `src/server/*` via `createServerFn`.
- PostgreSQL access lives in `lib/db/*` through Drizzle.
- Most client state is still Zustand under `store/*`. The issues page hydrates server data into `store/issues-store.ts` from `components/common/issues/all-issues.tsx`.

## Product Scope

- Treat the product as a personal issue tracker fork, not a team/workspace app.
- Do not reintroduce org, workspace, or team abstractions unless explicitly requested.
- `inbox` and much of `settings` are low-priority placeholder surfaces; avoid polishing them unless the task requires it.
- The current user model is intentionally single-user and lives in `lib/current-user.ts`.

## Data Reality

- Projects, issues, and labels already have PostgreSQL-backed flows; extend those instead of adding new mock-only behavior.
- The UI still depends on `mock-data/*` for presentation catalogs and some fallback options. If making the app more real, remove those dependencies deliberately instead of adding more.
- Local PostgreSQL is expected from the shared stack in `~/dev/postgres`.
- `DATABASE_URL` is required for Drizzle commands and server-side DB access. See `.env.example`.
